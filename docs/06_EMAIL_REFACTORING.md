# メール機能リファクタリング仕様

最終更新：2026-04-07 16:30

## 📋 概要

請求書アプリから Gmail 直接制御と SMTP 送信機能を削除し、スマホ標準のメールアプリに委ねる方針に変更しました。

## 🎯 変更理由

### 背景
- 既存実装が過度に複雑（`email_settings_screen.dart` は 979 行にも及んでいた）
- Google OAuth 設定の煩雑さから一般ユーザーにとって導入障壁が高かった
- TLS/証明書設定など、技術的ハードルが多すぎた

### 目的
- 依存関係を簡素化し、メンテナンス性を向上
- ユーザー体験を改善（スマホ標準アプリで十分機能）
- BCC 機能は取引先への送信控えとして必須のため維持

## 🔧 変更内容

### 削除された機能
2. **Gmail 連携機能（廃止）** ⚠️
    - Google Sign-In による直接送信
    - Gmail API を介した自動送付
    - テスト送信機能
    
    **廃止理由**:
    - ユーザーの動悸を考慮し、複雑な OAuth 設定を完全に排除
    - Gmail 連携による技術的ハードルが一般ユーザーにとって高すぎた
    - スマホ標準メールアプリで十分な UX を提供可能

3. **依存パッケージ**
   - `google_sign_in`（メール送信用）
   - `email_sender.dart` サービス
   - Gmail API 連携コード

### 維持された機能
1. **BCC 設定**
   - 送信控え用メールアドレス（複数カンマ区切り可）
   - 端末メールアプリの控え用として必須

2. **メールテンプレート**
   - メールヘッダーテンプレート（冒頭表示）
   - メールフッターテンプレート（末尾表示）

3. **「メールで共有」ボタン**
   - スマホ標準のメールアプリを起動
   - PDF を添付した状態で共有

## 🔄 動作フロー（変更後）

### 請求書送信プロセス
1. ユーザーが請求書詳細画面で「メールで共有」ボタンを押す
2. スマホ標準のメールアプリが起動
3. PDF が自動添付される
4. 宛先はユーザーが入力（BCC は設定画面で指定したアドレスに自動追加）
5. メールテンプレート（ヘッダー/フッター）が本文に自動挿入

### BCC の役割
- **送信控え**: 取引先にメールを送信した際、自分宛てにもコピーが送られる
- **記録管理**: 後から「いつ誰に何を送ったか」を確認可能
- **証拠保全**: 紛争時の証拠として残せる

## ⚙️ 設定画面（SM:メール設定）

### BCC 設定（必須）
```
自分のメールアドレスを入力。カンマ区切りで複数指定可能
例：me@gmail.com,backup@company.com
```

### メールテンプレート
- **ヘッダー**: メール本文の冒頭に表示
- **フッター**: メール本文の末尾に表示

## 📝 注意事項

1. **BCC は必須設定**: 未設定の場合、送信控えが取得できません
2. **Gmail 連携機能は廃止**: Gmail 同期機能は完全に削除されました。スマホ標準メールアプリのみ利用可能です。
3. **テンプレート管理**: メールヘッダー/フッターは引き続き利用可能
