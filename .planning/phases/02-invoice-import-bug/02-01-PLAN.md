<objective>
請求書インポート→保存→再表示の合計金額不一致バグを修正する

**バグの仕組み（要約）：**
1. `_performImport()` で請求書明細をインポート → `discountedUnitPrice = (item.subtotal / quantity).round()` で一度丸め
2. 表示時は `_calculate()` が `isFromInvoice=true` のため `originalSubtotal` を使用 → 正しい
3. 保存時 (`_save()`) も `originalSubtotal` を使用 → 正しい
4. 再読込時 (`_loadExisting()`) は `isFromInvoice=false` で復元し、`unitPrice = (subtotal / quantity).round()` で再度丸め
5. 再表示時は `_calculate()` が `isFromInvoice=false` のため `quantity * unitPrice` を使用 → 丸め誤差で合計不一致

**修正の方針：**
- `_LineItem` に `savedSubtotal` フィールドを追加（UI 状態のみ、DB 変更なし）
- `_loadExisting()` で DB から読み込んだ `DocumentItem.subtotal` を `savedSubtotal` に保持
- `_calculate()` と `_save()` で `savedSubtotal` を最優先で使用する
- `isFromInvoice` / `originalSubtotal` の分岐はそのまま維持（D-08 遵守）

**出力：** 修正された `lib/screens/sales_input_screen.dart`
</objective>
