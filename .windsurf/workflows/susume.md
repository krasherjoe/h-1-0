# 「進めてください」ワークフロー

このワークフローは `docs/AUTO_QUEUE.md` の先頭 ⏳ タスクを1件だけ処理する。
**1タスク＝1〜2ファイル・最大50行変更** を厳守してペナルティを防ぐ。

## ステップ

### Step 1: タスク確認
`docs/AUTO_QUEUE.md` を読み、先頭の `⏳` タスクを特定する。
- タスクID・対象ファイル・エラー内容を把握する
- タスクが見つからない場合は「全タスク完了」と報告して終了

### Step 2: 対象ファイルを読む
// turbo
対象ファイルを `read_file` で読む（必ず実行してから編集）

### Step 3: 修正を実装
// turbo
`edit` または `multi_edit` で修正する。
- **触るのは指定ファイルのみ（1〜2ファイル）**
- **1回の edit で変更できない場合は multi_edit を使う**
- 新しいロジックは追加しない。エラー・警告の解消のみ

### Step 4: 静的解析で確認
// turbo
```
flutter analyze [対象ファイルパス]
```
エラーが残っていれば Step 3 に戻って修正する（最大2回まで）。

### Step 5: git commit（日本語）
// turbo
```
git add [対象ファイルパス]
git commit -m "修正: [タスクID] [内容の要約]"
```

### Step 6: AUTO_QUEUE.md を更新
`docs/AUTO_QUEUE.md` の該当タスクを `⏳` → `✅` に変更し、完了日を記入する。

### Step 7: 次タスクをプレビュー
次の `⏳` タスクを読み上げて「次のタスク: [タイトル]」と報告する。

---

## 絶対ルール

| ルール | 理由 |
|--------|------|
| 1タスクで触るファイルは **最大2つ** | コンテキスト超過防止 |
| 変更行数は **最大50行** | 1ターン完結 |
| `flutter analyze [ファイル]` を **必ず実行** | エラー残存防止 |
| `git commit` は **必ず日本語メッセージ** | プロジェクトルール |
| タスクを自己判断で **追加・スキップしない** | キュー順序の保持 |
